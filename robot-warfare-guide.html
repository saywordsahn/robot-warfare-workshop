SECTION 00
HOW THE PROJECT IS ORGANIZED

Robot Warfare is a tower defense game built in Godot 4. Scarab robots march left across the screen. You click them to damage them, and place soldier turrets to defend your base. Here's the map of every file you'll be working with.

FILE WHAT IT DOES
scripts/main.gd The game brain — spawns scarabs, handles money, input, round timers, game over
scenes/enemies/scarab.gd Controls a single scarab: movement, health, taking damage, signals
scripts/soldiers/assault.gd Assault turret — targets the nearest robot in range
scripts/soldiers/sniper.gd Sniper turret — targets the robot with the most health
scripts/soldiers/machine_gunner.gd Machine gunner — shoots a random robot in range
scripts/globals/targeting.gd Autoload — shared functions for finding enemies (nearest, strongest, random)
scripts/globals/sound_manager.gd Autoload — plays sound effects and music from anywhere
scripts/util/damage_numbers.gd Floating damage numbers that rise and fade out
scripts/effects/small_fragments.gd Particle animation that plays when you click to shoot
Autoloads are special scripts that are always running. In this project: SoundManager, Targeting, and Game. Any script in the whole project can call them by name — no need to find the node first.
SECTION 01
THE GENERAL FLOW — START HERE

Run the game first. Understand what it does before changing anything. Here is the full flow from start to game over, so you always know where you are in the code.

01
TITLE SCREEN LOADS scenes/title.tscn is the first scene. Its script (scenes/title.gd) has two buttons: Start and Quit. Pressing Start calls get_tree().change_scene_to_file("res://scenes/main.tscn").
02
MAIN SCENE LOADS main.gd runs. Variables like money, round, and game_over are set to their starting values. Two timers start: ScarabSpawnTimer and RoundTimer.
03
SCARABS SPAWN Every time ScarabSpawnTimer fires, _on_scarab_spawn_timer_timeout() runs — it creates a new Scarab, places it on the right edge at a random Y position, and connects its signals.
04
SCARABS MOVE IN _process() Every frame, each scarab subtracts speed * delta from its X position. When X drops below 0, it emits on_base_reached.
05
ROUNDS GET HARDER Every time RoundTimer fires, the round counter goes up and ScarabSpawnTimer.wait_time is multiplied by 0.9 — making scarabs spawn 10% faster each round.
06
GAME OVER When a scarab reaches X < 0, on_base_reached signal fires → base_reached() in main runs → shows the game over label.
SECTION 02
THE SCARAB — YOUR FIRST ENEMY

The scarab is an Area2D node. This lets it detect mouse clicks. Open scenes/enemies/scarab.gd and read along. 

extends Area2D

# Signals tell other parts of the game what happened
signal on_killed
signal on_base_reached
signal on_attacked(amount: int, position)

# @export makes these editable in the Inspector!
@export var speed = 30
@export var max_health: int = 14
var health = max_health
var alive = true

func take_damage(amount: int):
    health -= amount
    on_attacked.emit(amount, global_position)
    if health <= 0:
        $AnimatedSprite2D.play('die')
        on_killed.emit()
        remove_from_group("robots")
        alive = false
        set_process(false)

func _input_event(viewport, event, shape_idx):
    if alive and Input.is_action_just_pressed("main_weapon"):
        take_damage(7)

func _process(delta: float) -> void:
    position.x -= speed * delta
    if global_position.x < 0:
        on_base_reached.emit()
WHAT IS DELTA?

delta is the time in seconds since the last frame. It's usually around 0.016 (for 60fps). Multiplying speed by delta makes movement frame-rate independent — the scarab moves the same real-world distance per second whether the game runs at 30fps or 120fps.

Without delta: fast computers = fast scarabs. With delta: everyone gets the same speed.

WHY remove_from_group("robots")?

Soldiers find targets using get_tree().get_nodes_in_group("robots"). If a dead scarab stays in the group, soldiers might keep shooting a corpse. Removing it from the group means dead robots are instantly ignored by all targeting code.

PRACTICE — SCARAB STATS

Select a Scarab in the scene. In the Inspector, find speed and max_health. Change them and run the game. What happens if speed is 200? What if max_health is 100?

You can't change @export variables while the game is running in release mode. But in the Editor, they're live-editable in the Inspector panel.
SECTION 03
SIGNALS — HOW NODES TALK TO EACH OTHER

Signals are how different parts of your game communicate without needing direct references to each other. The scarab doesn't know or care what happens when it dies — it just broadcasts the signal. Main listens and reacts.

STEP 1 — DEFINE THE SIGNAL

In scarab.gd, at the top:

signal on_killed

This creates a signal called on_killed that belongs to the scarab.

STEP 2 — EMIT THE SIGNAL

When something happens, fire it:

on_killed.emit()

This broadcasts "something happened" to anyone listening.

STEP 3 — CONNECT THE SIGNAL

In main.gd, after creating a scarab:

scarab.on_killed.connect(_on_enemy_killed)

Now when on_killed fires, _on_enemy_killed() runs.

STEP 4 — THE FUNCTION RUNS
func _on_enemy_killed():
    money += 10
    $UI/MoneyLabel.text = '$ ' + str(money)
The scarab also sends data with its on_attacked signal: on_attacked.emit(amount, global_position). The connected function must accept those same parameters: func add_damage_nums(amount, position):

SECTION 04
UI — LABELS, CANVAS LAYERS, AND VISIBILITY

The UI lives on a CanvasLayer node. This keeps it fixed on screen even when the camera moves. Labels and buttons live inside it.

UPDATING A LABEL

Labels have a text property. To change it from code:

$UI/MoneyLabel.text = '$ ' + str(money)
WHY str()?

money is an integer. Label text must be a String. str() converts any value into a String. Without it, Godot will give you an error.

SHOWING AND HIDING NODES

# Show a hidden node
$UI/GameOverLabel.show()

# Hide a visible node
$UI/GameOverLabel.hide()

In the Scene tab, you can also click the eye icon next to any node to hide it. Hidden nodes are invisible but still exist in code.

PRACTICE — ADD A ROUND LABEL

There is already a RoundLabel in the scene. In main.gd, find _on_round_timer_timeout(). It already updates the label. Read it and understand it. Then try: what would you change to show "WAVE 1" instead of "Round 1"?

The text is just a String. Change 'Round ' to whatever you want.
SECTION 05
INPUT — MOUSE AND KEYBOARD

Input actions are defined in Project → Project Settings → Input Map. This project has: main_weapon (left click), place_soldier (right click), pan_left / pan_right (A/D), pause (Escape), and slot_1/slot_2/slot_3 (keys 1/2/3).

CHECKING INPUT IN _process()

func _process(delta: float) -> void:
    if Input.is_action_just_pressed("main_weapon"):
        # runs once when the button is first pressed
        do_something()
is_action_just_pressed

Fires once on the frame the button is pressed. Good for: shooting, placing objects, opening menus.

is_action_pressed

Fires every frame while held. Good for: movement, charging, holding a fire button.

GETTING MOUSE POSITION

get_global_mouse_position()  # returns a Vector2 (x, y)

This is used in main.gd to place soldiers exactly where you right-click:

soldier.global_position = get_global_mouse_position()

_input_event ON AREA2D NODES

Scarab uses a different approach. Because it's an Area2D, it can receive its own input events when you click directly on it:

func _input_event(viewport: Viewport, event: InputEvent, shape_idx: int):
    if alive and Input.is_action_just_pressed("main_weapon"):
        take_damage(7)
Print the event parameter to see what data a click gives you: print(event). Try it — you'll see button index, position, and more.

THE 3-STEP INSTANCE PATTERN

Creating any object in Godot always follows this pattern:

# 1. Create the instance from the preloaded scene
var thing = THING_SCENE.instantiate()

# 2. Set properties on it
thing.global_position = get_global_mouse_position()

# 3. Add it to the scene tree so it actually appears
add_child(thing)

DAMAGE NUMBERS

When a scarab is hit, it emits on_attacked(amount, position). Main catches this and creates floating damage numbers:

func add_damage_nums(amount, position):
    var damage_numbers = DAMAGE_NUMBERS.instantiate()
    damage_numbers.global_position = position
    damage_numbers.set_amount(amount)
    add_child(damage_numbers)

The damage number itself moves up and fades:

func _process(delta: float):
    position.y -= delta * speed       # float upward
    modulate.a -= delta * fade_out_speed  # fade out

func _on_timer_timeout():
    queue_free()  # delete after timer expires

SECTION 07
SOLDIERS — THREE TURRET TYPES

All three soldier types share the same structure. The only difference is which targeting function they call. Compare them side by side:

SOLDIER TARGETING FUNCTION STRATEGY COST
Assault SLOT 1 Targeting.get_nearest_enemy() Shoots the closest robot in range $100
Machine Gunner SLOT 3 Targeting.get_random_enemy() Shoots a random robot in range $300
Sniper SLOT 2 Targeting.get_strongest_enemy() Shoots the robot with the most health $500

Each soldier's shoot timer calls the targeting function and, if it finds a target, calls take_damage() on it:

func _on_shoot_timer_timeout():
    var nearest = Targeting.get_nearest_enemy(global_position, range)
    if nearest != null:
        nearest.take_damage(damage)
        $MuzzleFlashAnimation.play('default')
        $MuzzleFlashAnimation.show()
WHAT IS null?

When a targeting function finds no valid target (no robots in range), it returns null — meaning "nothing". Always check if target != null before calling functions on it, or Godot will error.

SECTION 08
GLOBALS — TARGETING AND AUTOLOADS

Autoloads are scripts that are always running, accessible from anywhere by name. This project has three: SoundManager, Targeting, and Game. You never need to use $ or get_node() to access them.

TARGETING.GD

All four functions follow the same pattern — loop through every node in the "robots" group, check if it's in range, find the one matching the criteria:

func get_nearest_enemy(position, range):
    var nearest_enemy = null
    var nearest_distance = INF   # INF = infinity, every real distance is less

    for robot in get_tree().get_nodes_in_group("robots"):
        var distance = position.distance_to(robot.global_position)

        if distance <= range and distance < nearest_distance:
            nearest_enemy = robot
            nearest_distance = distance

    return nearest_enemy
THE "ROBOTS" GROUP

Each scarab scene has it added to the "robots" group in the Node → Groups panel. When a scarab dies, it's removed with remove_from_group("robots"). This is how all targeting functions know which nodes are valid targets.

FUNCTIONFINDSKEY VARIABLE IT TRACKS
get_nearest_enemy()Closest robot in rangenearest_distance
get_strongest_enemy()Robot with most healthhighest_health
get_random_enemy()Random robot in rangebuilds list, calls pick_random()
get_enemies_in_range()All robots in range (returns an array)appends to enemies array

SECTION 09
SOUNDS — THE SOUND MANAGER

SoundManager is an autoload in scripts/globals/sound_manager.gd. To play a sound from anywhere in the project, just call: 

SoundManager.play("gunshot")
SoundManager.play("lose")
SoundManager.play("ui-select")

Sounds are registered in a dictionary at the top of the file:

var sounds = {
    "gunshot": preload("res://sounds/effects/gunshot.mp3"),
    "ui-select": preload("res://sounds/ui/button-click.wav"),
    "lose": preload("res://sounds/effects/lose.wav")
}

ADDING A NEW SOUND

01
GET A SOUND FILE Find a free sound at freesound.org or generate one at sfbgames.itch.io/chiptone. Download as .wav or .mp3.
02
DROP IT IN THE PROJECT Drag it into the sounds/effects/ folder in Godot's FileSystem panel. Godot will import it automatically.
03
REGISTER IT Add a new entry to the sounds dictionary in sound_manager.gd: 
"explosion": preload("res://sounds/effects/explosion.wav")
04
PLAY IT Call it from anywhere: 
SoundManager.play("explosion")
Only one sound effect can play at a time with the current sfx_player setup. If two sounds fire at the same moment, the second one will cut off the first. You can fix this by creating more AudioStreamPlayer nodes for important sounds.
PRACTICE — ADD A DEATH SOUND

There's no sound when a scarab dies. Add one. You'll need to: find a sound file, add it to sound_manager.gd, then call SoundManager.play("death") inside take_damage() in scarab.gd when health <= 0.

ADVANCED

CREDITS SCROLL HARD

Make a credits screen that scrolls a Label upward using _process() or a Tween. Add a skip button. Connect it from the main menu. The label can include your name, your team's name, and "special thanks to" — whatever you want.

TARGETING: WEAKEST ENEMY HARD

Write get_weakest_enemy(position, range) in targeting.gd that targets the robot with the lowest current health. Create a new soldier type that uses it. Think about when this strategy is better or worse than targeting the strongest.

PAUSE MENU SOUNDS HARD

The pause menu works, but there are no sounds for opening or closing it. The catch: when the tree is paused, AudioStreamPlayer also stops. Look up process_mode on the SoundManager node — set it to "Always" so it keeps running while paused.
